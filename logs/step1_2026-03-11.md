## Design Decisions Made During Implementation

**CSV auto-detection**: The WEB Bible is available in several download formats from
ebible.org (USFM, readaloud CSV, plain CSV). Rather than hard-coding one format,
`step1_ingest_web.py` sniffs the first row and dispatches to one of four parsers
(A = 4-col OSIS, B = reference+text, C = ebible readaloud id, D = numbered books).
This avoids re-work if the download format changes.

**Alternate abbreviation table**: ebible.org uses slightly different book codes than
OpenBible (e.g. `EXOD` vs `EXO`, `JUDG` vs `JDG`). The `ALTERNATE_ABBR` dict in
`step1_ingest_web.py` handles about 30 known variants. If a new variant surfaces on
first run, add it there — one line, no logic change needed.

**Chapter ID synthetic key**: Neo4j constraints require a single unique property.
Chapters don't have a natural unique ID, so we synthesise `chapter_id = "{BOOK_OSIS}.{chapter}"`
(e.g. `"ROM.8"`).  This follows the same OSIS pattern as verse IDs and is stable
across re-runs.

**`char_count` computed at parse time**: The PRD calls for `char_count >= 25` filtering
in Step 3c.  Computing it during Step 1 ingest costs nothing extra and avoids a
separate UPDATE pass on 31K nodes later.

**Dry-run flag**: `--dry-run` parses the CSV and prints the first 10 rows with OSIS IDs
without touching Neo4j.  This is the right first action after downloading the WEB file —
confirm IDs like `ROM.8.28` and `PSA.23.1` appear correctly before any writes.

## Constraints Created by This Step

```cypher
CREATE CONSTRAINT verse_id_unique   IF NOT EXISTS FOR (v:Verse)   REQUIRE v.id         IS UNIQUE
CREATE CONSTRAINT book_name_unique  IF NOT EXISTS FOR (b:Book)    REQUIRE b.name        IS UNIQUE
CREATE CONSTRAINT chapter_id_unique IF NOT EXISTS FOR (c:Chapter) REQUIRE c.chapter_id  IS UNIQUE
```

These constraints are idempotent (`IF NOT EXISTS`) — safe to re-run.  Without them,
every `MERGE` in Step 2 degrades to a full table scan.

## Known Issues / Watch Points

- The WEB includes deuterocanonical books in some distributions (Tobit, Judith, etc.).
  The canonical book table has exactly 66 books.  If the CSV contains extra books,
  the parser will emit warnings (`unrecognised book abbreviation`) and skip those rows.
  This is expected behaviour for the Protestant 66-book canon.

- `John 11:35` ("Jesus wept.") has `char_count = 11`, well below the `MIN_CHAR_COUNT = 25`
  threshold.  It will be imported as a Verse node (Step 1 imports everything) but will be
  excluded from IS_SIMILAR edge creation in Step 3c.  This is by design.

- If Neo4j is not running, `utils/neo4j_conn.py` raises `ServiceUnavailable` immediately
  (the `verify_connectivity()` call).  Start Neo4j Desktop first.
